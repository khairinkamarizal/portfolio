Print infographics have different rules from digital ones. No hover states, no zoom capability, no progressive disclosure. Everything the viewer needs has to be present and legible in one static composition at one fixed size. That constraint demands rigorous design thinking.

## The Hierarchy of Data

Before any visual design, I establish what the data is actually saying. What is the key insight the infographic should communicate? What is secondary context? What is supporting detail?

This hierarchy drives the visual hierarchy. The key insight is the largest, most prominent element. Supporting detail is smaller and subordinate. If everything is the same visual weight, the viewer has no guidance about what matters most.

## Choosing the Right Chart Type

Different relationships in data call for different chart types, and using the wrong type obscures the data rather than revealing it. Part-to-whole relationships: pie charts or treemaps. Change over time: line charts or area charts. Comparison between categories: bar charts. Correlation between variables: scatter plots.

The mistake I see most often: using a pie chart for time-series data, or a 3D effect on any chart that adds visual complexity without adding information. Decoration that obscures data is worse than no decoration.

## Color in Data Visualization

Color in infographics serves two purposes: categorical differentiation (this element belongs to this group) and quantitative encoding (more of this value means darker or more saturated color).

I use color sparingly. Four to six categorical colors maximum before color becomes hard to distinguish. For sequential data (a range from low to high), a single-hue progression — from light to dark — is cleaner and more readable than a multi-hue range.

Every color combination in an infographic must pass contrast testing. Red-green differentiation fails for colorblind viewers (affecting about 8% of male readers). I check every infographic against a colorblind simulation.

## Typography in Data

Type in infographics serves different roles than type in editorial design. Axis labels need to be legible at very small sizes. Data labels need to be clearly associated with their data points. Headlines need to be immediately scannable.

I use a single typeface family for all infographic type, varying weight and size for hierarchy. Two typefaces in a data visualization creates visual noise that competes with the data.

## The Annotation Layer

The most useful thing a well-designed infographic can do is point out what to notice. An annotation — a callout, a highlighted element, an explanatory label — guides the viewer to the insight. Raw data without annotation asks the viewer to do all the interpretive work.

I add a minimum of two annotations to every significant infographic: one pointing to the key insight, one providing context for that insight. The annotation layer is where the editorial perspective lives.
