One of the most common points of failure in logo design projects is the delivery. The design is finalized, the client approves, and then they receive... one SVG file. Then six months later they're back asking for a version that works on a dark background, a version without the tagline, a version for embroidery, a version for the website favicon.

A complete logo delivery anticipates these needs and includes the full system upfront. This is better for the client and better for the designer, who doesn't spend months fielding requests for file variations.

## The Core Variations

Every logo needs at minimum these variations:

**Full lockup (horizontal).** The most complete version: mark, wordmark, and tagline if applicable. This is the default version for most applications.

**Full lockup (stacked).** The vertical arrangement for contexts where horizontal space is limited.

**Mark only.** The symbol without the wordmark. Used at large scale (signage, merchandise) and small scale (app icons, favicons, social profile images).

**Wordmark only.** The brand name set in the brand typography without the symbol. Useful for contexts where the symbol is already established and repetition isn't needed.

**Simplified version.** A version of the mark that works at very small sizes, with detail reduced for legibility. Not every mark needs this, but complex marks often do.

## The Color Variations

For each lockup configuration, provide these color versions:

**Full color.** The primary brand color version.

**Reversed (white).** The white version for use on dark backgrounds. Don't let clients figure out how to make a white version themselves from a colored file — they'll get it wrong.

**Single color black.** A black version for contexts where color isn't available.

**Single color white.** Same as above but white.

**Spot color version.** If the brand uses a specific Pantone color, a version built with that spot color for professional print use.

That's five color versions for each of four layout configurations. Twenty files minimum for a thorough delivery.

## The File Formats

**SVG (vector, screen).** The master format for web and digital use. Scales to any size without quality loss.

**EPS or AI (vector, print).** The master format for professional print production. Printers prefer EPS for its broad compatibility; AI files preserve editability for future modifications.

**PNG (raster, transparent background).** A convenience format for non-design applications: presentations, word processing documents, email signatures. Provide at multiple sizes: 1x, 2x, and a large 3000px version.

**PDF (vector, universal).** A universal vector format that opens in any PDF viewer. Useful for clients who need to share files with vendors who don't have Illustrator or Figma.

**ICO or favicon set.** If the brand has a digital presence, provide the favicon files specifically: 16x16, 32x32, 180x180 (Apple touch icon), and ideally an SVG favicon.

## The Organization

I deliver logo packages as a structured ZIP archive with a clear folder hierarchy:

```
BrandName_LogoPackage/
  01_Primary_Lockup/
    01_FullColor/
    02_Reversed/
    03_BlackWhite/
  02_Stacked_Lockup/
    ...
  03_Mark_Only/
    ...
  04_Wordmark_Only/
    ...
  README.txt
```

The README explains which file to use in which context. Clients shouldn't have to guess whether they need an EPS or a PNG for a specific application.

## The Geometry Guide

For marks with construction geometry, I include the geometry guide as a PDF. This allows future designers, fabricators, or vendors to reconstruct the mark accurately if needed. It also demonstrates the rigor of the design process.

A complete logo package delivered at the end of a project is a professional courtesy and a practical service. It reduces ongoing support requests and ensures the brand is applied correctly long after the design engagement ends.
