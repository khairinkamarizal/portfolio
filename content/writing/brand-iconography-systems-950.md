## Icons Are Not Afterthoughts

Most brand identity projects produce a logo system, a color palette, and typography guidelines. Then, six months later, someone needs icons for a website, a presentation, or a brochure, and they download whatever looks close from a stock icon library.

The result is a brand that looks considered in its primary identity and generic in its secondary communications. The icon set becomes a visual wildcard that competes with the designed identity rather than extending it.

I build icon systems as part of brand identity projects, not as separate afterthoughts.

## What Makes an Icon Set Branded

A branded icon set has a consistent visual language that derives from the identity system. The stroke weight relates to the typeface's stroke weight. The corner radius relates to the brand mark's edge treatment — sharp for brands with angular marks, rounded for brands with curved marks. The level of detail relates to the overall aesthetic register — simplified and geometric for minimalist brands, more detailed and expressive for brands with richer visual personalities.

When all of these decisions are consistent, the icons feel like they belong to the brand. When they're inconsistent — even if each individual icon is competently drawn — they look assembled rather than designed.

## The Grid Foundation

I build every icon system on a consistent grid. A 24×24 or 32×32 pixel grid is standard, with an optical margin — typically two units — inside the bounding box. Every icon in the set uses the same grid, the same optical margin, and the same live area.

The grid isn't just a technical tool. It's what creates the sense that all the icons in the set share a common scale and weight relationship. Without a grid, icon sets drift in perceived size and heaviness even if they're nominally the same pixel dimensions.

## Line Icons vs. Filled Icons

The choice between outlined (line) icons and filled (solid) icons should be driven by brand character, not trend.

Line icons feel lighter, more precise, and more sophisticated. They work well for brands that value negative space and a clean, editorial quality.

Filled icons feel heavier, more accessible, and more functional. They're easier to read at small sizes and communicate more confidently in dense interfaces or print contexts.

Some brands use both — outlined icons for decorative or supporting contexts, filled icons for primary functional contexts. If using both, the visual language still needs to be consistent: the same stroke weight in the outlined icons, the same corner radius treatment in both.

## How Many Icons Is Enough

I start every icon project by auditing the brand's actual use cases. What categories of things does this brand need to communicate iconically? Navigation categories, service areas, product types, actions, social media platforms. I derive the icon set from this audit rather than producing a generic "essential icons" package.

Most brands need between twenty and sixty icons for comprehensive coverage of their primary communications. More than that usually indicates either scope creep or a desire to replicate a general-purpose icon library rather than a brand-specific one.

## Delivering the Icon System

Icons are delivered as SVG files for flexibility and as PNG exports at standard sizes (16px, 24px, 32px, 48px). I also deliver an organized icon library in whatever design tool the team uses — Figma, Sketch, or Adobe XD — so the icons are immediately accessible in the brand's design environment.

Documentation covers naming conventions (how to find the right icon), usage rules (which icons for which contexts), and the visual language principles so future icons can be drawn consistently by any designer who follows the guidelines.
