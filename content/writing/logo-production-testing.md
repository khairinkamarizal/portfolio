I've seen brilliant logos destroyed by production. A mark that looked stunning in the presentation PDF, embossed on a business card or embroidered on a polo, turns into an unrecognizable smear. The logo failed not because the design was wrong, but because it was never tested against the conditions it would actually live in.

Testing for production isn't a final step — it's part of the design process.

## The Production Conditions Checklist

Every logo I design gets tested against a standard set of conditions before I consider it finished:

**Single color:** Black only, no gradients. If the logo doesn't work in pure black, it doesn't work. Every other color condition is a variation on this foundation.

**Reversed:** White on a dark background. Some logos that work perfectly in black fall apart reversed — thin strokes disappear, fine details merge, the mark loses its character.

**Small scale:** The favicon test (16x16px or 32x32px) is extreme, but it reveals whether the mark has a viable simplified version. At this size, most complex marks need a simplified "icon" variant.

**Embroidery simulation:** Embroidery has a minimum stitch count, a maximum level of detail, and its own aesthetic qualities. I view logos through an embroidery simulation filter (or send test files to an embroidery digitizer) for any brand that will put the logo on apparel.

**Screen print:** One-color, limited-color screen printing is similar to embroidery — minimum detail, simplified forms, no gradients.

**Embossing/engraving:** Blind emboss and laser engraving require clean outlines with no interior detail that can't be rendered as a single depth.

## Building the System to Survive These Tests

The insight from this testing process is that logo design is really a system design problem. You're not designing a single mark — you're designing a family of related marks that all look like the same thing in different contexts.

The primary logo might be a full-color lockup with a symbol and wordmark. The secondary might be a simplified single-color version. The tertiary might be a standalone icon variant. The favicon might be an even further reduced geometric abstraction.

All of these need to feel like the same brand, even as they vary significantly in detail and complexity.

## The Files Are Part of the Design

Clean, well-organized vector files are as important as the visual design. A logo package with messy paths, rasterized elements embedded in an SVG, or unoutlined fonts is an unusable logo package regardless of how good the design looks.

I deliver logos with properly outlined type, clean paths with minimal anchor points, named color swatches, and a clearly organized file structure. The production artist who receives these files should be able to use them immediately without reconstruction.
