## Every Animation Is a Story

When elements appear on screen in a motion piece, they don't all arrive at once. They arrive in sequence. That sequence is a story — a directed experience where the viewer is guided through information in a specific order.

Most of the motion design I see doesn't think about this carefully enough. Elements appear because the designer built them in that order, not because the order was intentionally designed for the viewer's comprehension.

Staging — the deliberate choreography of when and how elements appear — is one of the most underused tools in motion design.

## The First Frame Problem

The first thing that appears on screen sets expectations for everything that follows. If a complex background texture is the first element in, the viewer's eye has to work to find the subject. If a clear, strong primary element is first, the viewer is oriented immediately.

I always design the first element reveal first. What is the most important piece of information? That's what arrives first. Everything else supports it or builds on it.

## Building From Foundation to Detail

My default staging structure moves from general to specific, from large to small:

First, the structural or spatial context — the background, the container shape, the compositional frame.

Then the primary element — the most important piece of information, the headline, the main message.

Then supporting elements — secondary information, descriptive text, supporting imagery.

Finally, details — fine print, tertiary information, decorative elements.

This structure means the viewer always has context for what arrives next. The primary message lands in an established space. The supporting information arrives after the viewer has understood the main point.

## Staging for Emphasis

Staging can be used to create emphasis through isolation. If the primary message arrives and holds alone on screen for even half a second before anything else appears, that isolation creates weight. The viewer has only one thing to look at, so they look at it completely.

This is one of my favorite techniques for headlines: bring the text in, let it breathe alone for 400–600ms, then bring in the supporting context. The result is a headline that feels authoritative rather than just large.

## The Cascade and When to Use It

A cascade — multiple elements staggering in close succession — creates energy and momentum. It works well for:

- Lists where the parallel structure is part of the message
- Charts or diagrams building element by element
- Credits or information sequences

What it doesn't do well: communicate hierarchy. When everything cascades at the same rate, the viewer doesn't know what matters more. Use cascade within a single tier of information, not across tiers.

## Exits Matter as Much as Entrances

Most designers think carefully about how elements appear and carelessly about how they leave. But exits are the last impression of each element. A graceful exit preserves the quality of what came before. An abrupt exit undercuts it.

I design exit choreography with the same care as entrance choreography. Usually this means reversing the entrance staging: details exit first, primary elements exit last. The last thing the viewer sees before the screen clears should be the most important thing — because that's what they'll remember.

## The Rule I Follow

Every element gets a reason to appear when it appears. If I can't articulate why an element appears at that specific moment in the sequence, I haven't designed the staging — I've just built it.

Intentional staging turns a series of animated elements into a directed experience. That's the difference between motion design and just things moving.
