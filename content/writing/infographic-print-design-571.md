## The Static Information Challenge

Digital infographics can use hover states, expandable sections, tooltips, and zoom. Print infographics have none of this. Every piece of information must be legible at the size it's printed, in a single static state.

This constraint eliminates lazy approaches to information density. You can't hide complexity behind an interaction. Everything the viewer needs to understand must be visible and clear at once.

## Hierarchy in Data Visualization

The most common failure in print infographics is treating all information as equally important. The visualization becomes a field of equally weighted data points without a clear story.

Good print infographic design has a hierarchy: the main finding is emphasized, supporting data is clear but secondary, contextual information is present but receding.

I establish this hierarchy through scale, color contrast, and position. The primary finding is largest, highest contrast, most prominently positioned. Supporting data is smaller, lower contrast, positioned as evidence for the main finding. Annotations and source notes are smallest and most receding.

## Chart Type Selection

The chart type should match the type of comparison being made:

**Comparison over time** — line charts and area charts. They show temporal trend clearly.

**Comparison between categories** — bar charts (horizontal for long category names, vertical for time-based categories).

**Part-to-whole relationships** — pie charts for simple compositions, treemaps or stacked bars for complex ones.

**Correlation between variables** — scatter plots.

**Distribution** — histograms, box plots.

The most common mistake: using pie charts for more than four or five categories, or for any data where the differences between segments are subtle.

## Simplification as Design Work

Every default chart produced by data visualization software requires significant simplification before it's ready for print. Default charts include:

- Gridlines that compete with the data
- Legend boxes that take up space and require the eye to travel back and forth to decode the chart
- Data labels that repeat information already shown in the axes
- Default color palettes that have no relationship to the brand

I remove all of these defaults and add back only what's necessary. Gridlines: removed or reduced to subtle guides. Legends: replaced with direct labels on the data wherever possible. Redundant labels: removed. Color: replaced with the brand palette.

## Type in Infographics

Typeface, size, and hierarchy in an infographic should be consistent with the brand's typographic system, not whatever the chart software defaulted to.

Chart titles are headings in the brand's heading typeface. Data labels are body text in the brand's body typeface. Source notes are the smallest text size in the system.

Getting the typography right in an infographic is one of the things that separates professional information design from something produced with default settings.

## Testing Before Printing

I print test proofs of complex infographics before finalizing. What looks legible on a high-resolution monitor at full zoom can be completely illegible at print size.

Specifically testing: the smallest text in the visualization, the finest lines in any chart, the legibility of color-coded elements in single-color or low-color contexts.

If any element fails at print size, I resize or simplify until it doesn't. The print state is the final state. Design for it.
