## Infographics are communication, not decoration

The worst infographics are beautiful decorative objects that communicate nothing. The chart is elaborate but unreadable. The icons are charming but the data is inaccessible. The color palette is on-brand but the contrast between data series is inadequate. Someone spent a lot of time making it look interesting and very little time making it work.

My rule for infographic design: every visual decision must serve the data. If a design element doesn't make the data clearer, it shouldn't be there.

## The data story first

Before I design any infographic, I identify the data story — the specific insight or comparison that the infographic is meant to communicate. Not "here is all the data we have about this topic," but "here is the one thing we want the viewer to understand after looking at this."

Most data sets contain multiple potential stories. My job is to choose the most important one and design the infographic to tell that story clearly and unmistakably. An infographic that tries to tell five stories tells none of them.

## Chart type is not an aesthetic choice

The chart type should be determined by the data structure, not by visual preference. Bar charts for comparisons across categories. Line charts for changes over time. Scatter plots for correlations. Pie charts for part-to-whole relationships with a small number of segments.

Using a visually interesting chart type that's wrong for the data structure is one of the most common infographic failures. A radar chart looks sophisticated, but for most data it communicates far less clearly than a simple bar chart. The goal is comprehension, not visual novelty.

## Color for data, not decoration

Color in data visualization has a specific job: to differentiate data series from each other and to encode meaning (positive/negative, high/low, category membership). Color used for decoration — making the chart look more colorful — actively harms legibility by creating false visual relationships.

I use color in infographics with maximum restraint. A single accent color to highlight the key data point. Two distinct colors to differentiate two data series. A sequential color scale with adequate contrast for continuous data. No more than what's needed for the data structure.

## Text in data visualization

Labels, legends, axes, and annotations are as important as the visual elements in an infographic. They're also consistently treated as afterthoughts — set too small, in typefaces that don't work at small sizes, positioned awkwardly.

I give infographic typography the same attention I give headline typography. The data label at 8pt needs to be legible. The axis labels need to communicate their scale immediately. The annotation pointing to the key data point needs to be clear and directly connected to what it's labeling.

## The print resolution requirement

Infographics for print need to be built at print resolution — 300 DPI at final print size. Charts created in spreadsheet software (Excel, Google Sheets) and exported as screen-resolution images will look terrible in print. I rebuild charts that need to go to print in vector tools (Illustrator, Figma) or in chart libraries that export vector formats.
