I've taken over brand identity projects where the client has been using a typeface they don't actually have the license to use. Sometimes they bought one license for one designer and everyone has been installing it. Sometimes a designer provided the font files without the license documentation. Sometimes the client doesn't realize that the free version of a typeface they downloaded isn't licensed for commercial use.

Font licensing is not a bureaucratic technicality. It's a genuine legal and ethical requirement, and getting it wrong can have real consequences.

## The Types of Font Licenses

Font licenses vary significantly by foundry, but the major categories are:

**Desktop licenses:** Permit installation on a specific number of computers and use in print documents, PDFs, and other static output. The most common license type. Typically priced per font weight, with volume discounts.

**Webfont licenses:** Permit use in CSS @font-face declarations, typically priced by monthly page views. Different from desktop licenses — you can't use a desktop-licensed font on the web and vice versa, except with licenses that bundle both.

**App/ePub licenses:** Permit embedding in applications, e-books, or other digital products. Separate from webfont licensing.

**Broadcast/motion graphics licenses:** Permit use in video, broadcast, and motion graphics. Some foundries charge separately for this use case; others include it in desktop licenses.

**Server-side/OEM licenses:** For environments where the font is processed or distributed at scale — document generation systems, apps that embed fonts in PDFs on the server side.

## What a Brand Identity Needs

When I specify typography for a brand identity, I list what licenses the client will need based on their actual use cases. A company that uses the brand typeface in:

- Print collateral (business cards, brochures, signage) → Desktop license
- Their website → Webfont license  
- Email marketing → Typically covered by webfont or desktop license; check the specific foundry
- Video and motion graphics → Broadcast license (sometimes included, sometimes separate)
- A customer-facing application → App license

Many clients need all of these. I make sure they know that before they're committed to a typeface.

## Variable Fonts and Newer License Structures

Variable fonts — which contain a range of weights and styles in a single file — have simplified some licensing questions but complicated others. Some foundries price variable fonts as a single license that covers the full range; others price each instance separately.

I note the specific file format and license structure for every typeface I specify, because variable font licenses vary significantly between foundries.

## Free Doesn't Mean No Restrictions

Google Fonts and other free type resources have made high-quality typefaces freely available with open licenses. This is genuinely useful and I recommend Google Fonts typefaces regularly for clients who need broad deployment flexibility without ongoing licensing costs.

But "free" doesn't mean "no restrictions." Google Fonts typefaces are licensed under the SIL Open Font License or Apache License — both permissive but with specific terms. I read the license for every typeface I specify, even the free ones.

## Recommending Fonts With Practical Licensing

For clients who are just starting out or have limited budgets, I weight my typeface recommendations toward faces with straightforward, inclusive licensing. A typeface that requires four separate license purchases for different use cases is a real ongoing operational burden. A face with a single license that covers all digital and print use simplifies everything.
