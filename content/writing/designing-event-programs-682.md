Event programs — the printed schedules distributed at concerts, theater performances, festivals, conferences — are usually the last thing that gets designed in an event production schedule. By the time they're commissioned, the deadline is days away, the budget is thin, and the content is still arriving in pieces.

The result, typically, is a program that reads as an obligation rather than an experience. Generic layout, whatever typeface was available, content crammed in at whatever size fits.

The programs that work — the ones attendees keep, read during the show, bring home — are the ones that were designed with the same care as the event poster. They're part of the experience.

## Format Is a Design Decision

The format of the program — its size, folding structure, page count — is a design decision, not just a production convenience. A standard A5 booklet for a jazz concert communicates something different than a single-fold A4 sheet for a theater performance, which communicates something different than a folded broadsheet for a festival.

The format should be appropriate to the event, practically usable (can you hold it comfortably in the dark while watching a performance?), and consistent with the event's visual identity. I brief the format before I brief the content layout.

## Information Architecture

Event programs have a predictable information architecture, but executing it well requires hierarchy decisions:

**Essential information (read before the event):** Artist/performer lineup, schedule, venue map if needed, pre-event notes from curators or directors.

**Contextual information (read during or after):** Biographical notes on performers, essay or curatorial text, notes on specific works.

**Supporting information (reference):** Credits, sponsors, thank-yous, contact and subscription information.

These three tiers need visual differentiation that communicates their status. The essential information should be immediately findable. The contextual content should be readable and engaging. The supporting information should be present but clearly subordinate.

## Typography for Low-Light Reading

Programs are often read in low light — a theater, a concert hall, an outdoor evening event. This is a legibility constraint that needs to be designed for, not ignored.

Low-light reading increases the importance of contrast, typeface legibility, and type size. The minimum comfortable size for body text in a low-light reading context is larger than for bright-light reading — I go to 10pt as a floor for any text that needs to be read in the dark.

White or light type on a dark background can work well in low-light contexts because the self-illuminated quality of a high-contrast reversal is more visible in dim conditions. But reversed type requires careful weight management, as already discussed.

## The Program as Keepsake

For events that want to create a more premium experience, the program can be designed as an object worth keeping. Better paper stock, higher-quality print finish, format or proportion that's distinctive. A program that feels like an artifact of the event extends the experience beyond the evening.

This is a budget conversation, but it's also a brand conversation. What does this event want to be remembered as? The program is part of the answer.
